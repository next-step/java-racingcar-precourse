# 자동차 경주 게임
## 학습 목표 (주관적인)
* TDD 학습
  * 테스트 케이스를 작성 후 실패, 성공, 리펙토링 과정을 반복하며 구현체를 작성
  * 테스트 케이스를 통과할 수 있는 최소한의 코드로 구현 (예측하여 over engineering 하지말 것)
* Domain 기반으로 설계
* 일급콜렉션 활용
* 원시값과 문자열 포장 활용
* 형상관리
  * 기능 단위로 테스트 완료 후 commit
  * docs/feat/test/refactor/fix 등의 작업을 동시에 진행 후 commit 하지 않는다.
  * 기능 개발 중에도 기능 목록을 좀더 명확하고 세부적으로 나눠 현행화한다.

## 요구사항
* 주어진 횟수 동안 n대의 자동차는 전진 또는 멈출 수 있다.
* 각 자동차에 이름을 부여할 수 있다. 전진하는 자동차를 출력할 때 자동차 이름을 같이 출력한다.
* 자동차 이름은 쉼표(,)를 기준으로 구분하며 이름은 5자이하만 가능하다.
* 사용자는 몇번의 이동을 할 것인지를 입력할 수 있어야 한다.
* 전진하는 조건은 0에서 9사이에서 random 값을 구한 후 random 값이 4이상 일 경우 전진하고, 3이하의 값이면 멈춘다.
* 자동차 경주 게임을 완료한 후 누가 우승했는지를 알려준다.우승자는 한명이상 일 수 있다.

## 기능목록
* 자동차 이름 입력
  * 쉼표(,)를 기준으로 구분
  * 이름은 5자 이하
  * 이름 중복 체크
* 자동차 이동 횟수 입력
  * 0보다 큰 숫자
* 실행 출력 (횟수별)
  * 자동차 이름 표기
  * 자동차 전진 횟수 표기 (ex: apple:---)
  * 0~9 사이 random 값이 4이상이면 전진, 아닐 경우 멈춤
* 결과 출력
  * 가장 많이 이동한 자동차들을 모두 출력

## 구현목록
### 구조
* Controller
  * View, Domain 상호 간의 값을 전달
* Domain
  * 차동차
  * 자동차이름(사용자 입력)
  * 자동차위치
  * 자동차목록
  * 이동횟수(사용자 입력)
  * 이동상태
  * 랜덤코드
* View
  * 사용자 입력 view (자동차이름, 횟수)
  * 결과 출력 view

### 테스트 케이스
* RandomNumber 생성
```
0~9 사이 랜덤한 숫자 생성
```

* CarName 생성
```
사용자에게 입력 받은 문자열을 5자리 이하인지 체크
유효하지 않을 경우 오류 발생
```

* CarPosition 생성
```
최초 생성 시 기본 값 0으로 설정 후
이동 명령이 있을 경우 1씩 증가
```

* Car 생성
```
CarName, CarPosition을 생성
Cars에서 Collection의 contains 사용을 위해 equals, hashCode 구현

입력 -> 결과
--------------------
페라리 -> 생성
에스턴마틴 -> 생성
레이싱포인트 -> 오류
```

* Car move 구현
```
RandomNumber를 통해 MoveStatus 판별하여
자동차 이동 메서드에 전달하여 CarPosition을 증가시키거나 유지시킨다.
```

* Cars 생성
```
사용자에게 입력 받은 문자열을 ','로 구분하여
각각의 Car를 생성 후 Cars에 내부 콜렉션에 추가한다.
```

* Cars move 구현
```
Cars의 내부 콜렉션에 접근하여 Car move를 각각 실행한다.
위 과정을 사용자에게 입력 받은 실행 횟수 만큼 반복 실행한다.
```

* Cars 우승자 목록 필터링
```
Cars move 완료 후 콜랙션에 포함된 Car 중에 가장 값이 큰 CarPosition을 구하고
해당 CarPosition이 동일한 Car를 조회해 새로운 Cars를 생성 후 반환한다.
```

* 이동숫자 입력
```
사용자에게 입력 받은 문자열을 숫자로 변환

입력 -> 결과
--------------------
"" -> 오류
a1a -> 오류
0 -> 오류
10 -> 10
1 -> 1
```
