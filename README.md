# 자동차 경주 게임
## 진행 방법
* 자동차 경주 게임 요구사항을 파악한다.
* 요구사항에 대한 구현을 완료한 후 자신의 github 아이디에 해당하는 브랜치에 Pull Request(이하 PR)를 통해 과제를 제출한다.

## 과제 제출 과정
* [과제 제출 방법](https://github.com/next-step/nextstep-docs/tree/master/precourse)

## 기능 요구사항
* 주어진 횟수 동안 n대의 자동차는 전진 또는 멈출 수 있다.
* 각 자동차에 이름을 부여하고, 전진하는 자동차를 출력할 때 자동차 이름을 같이 출력한다.
* 자동차 이름은 쉼표(,)를 기준으로 구분하며 이름은 5자 이하만 가능하다.
* 사용자는 몇번의 이동을 할 것인지를 입력할 수 있어야 한다.
* 전진하는 조건은 0~9 랜덤 값을 구한 후 4 이상일 경우 전진, 3 이하일 경우 멈춘다.
* 자동차 경주 게임을 완료한 후 누가 우승했는지 알려준다.
* 우승자가 한 명 이상일 경우 쉼표(,)로 이름을 구분해 출력한다.
* 사용자가 잘못 된 값을 입력할 경우 "[ERROR]"로 시작하는 에러 메시지를 출력 후 입력을 다시 받는다.

## 프로그래밍 요구사항
* 자바 코드 컨벤션을 지키면서 프로그래밍한다.
* 인덴트 뎁스를 2가 넘지 않도록 구현한다. 1까지만 허용한다.
* 자바 8에 추가된 stream api를 사용하지 않고 구현해야 한다. 단, 람다는 사용 가능하다.
* else 예약어를 쓰지 않는다. switch/case 도 허용하지 않는다.
* 함수(또는 메소드)의 길이가 10라인을 넘어가지 않도록 하고 함수가 한가지 일만 잘 하도록 구현한다.
* 일급콜렉션을 활용해 구현한다.
  - 콜렉션을 포함한 클래스는 반드시 다른 멤버 변수가 없어야 한다.
* 모든 원시값과 문자열을 포장한다.
* 도메인 로직에 단위 테스트를 구현해야 한다. 단, UI 로직은 제외

## 구현할 기능 목록
* 사용자에게 경주할 자동차 이름을 입력받는다. (단, 차가 여러대일 경우 쉼표로 구분)
  * 쉼표를 구분자로 경주할 자동차 목록을 만든다.
  * 이름이 5자 이하인가?
* 사용자에게 진행할 이동 횟수를 받는다.
  * 1 이상의 숫자인가?
* 0부터 9까지 랜덤 값을 구한다.
  * 4 이상일 경우 전진하고, 3 이하의 값이면 멈춘다.
* 실행 결과를 자동차별로 이름과 함께 출력한다.
  * 전진인 경우 출력 시 '-'를 추가하여 사용자에게 보여준다.
* 최종 우승자를 출력한다. 전진 횟수가 동일한 경우 쉼표로 구분하여 출력한다.
