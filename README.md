# 자동차 경주 게임
## 진행 방법

---
* 숫자 야구 게임 요구사항을 파악한다.
* 요구사항에 대한 구현을 완료한 후 자신의 github 아이디에 해당하는 브랜치에 Pull Request(이하 PR)를 통해 과제를 제출한다.

## 과제 제출 과정

---
* [과제 제출 방법](https://github.com/next-step/nextstep-docs/tree/master/precourse)

## 기능 개발리스트 

--- 

* 레이싱카에 대한 기능 (Car)
    * 레이싱카는 이름을 받아서 생성한다.
    * 레이싱카는 이름(CarName) 과 위치(Position) 에 대한 정보를 가진다.
    * 전략에 따라 전진여부를 정한다. (기본값 랜덤)

* 레이싱카 이름 (CarName)
  * 5글자 이하다.
  * 문자열을 받아 생성한다.
  
* 위치에 대한 기능 (Position)
  * 초기값은 1이다.
  * 위치값을 1씩 증가 할 수 있다.
  * 초기값보다 작은 값은 설정할 수 없다.
  * Position 끼리 비교가 가능해야 한다.

* 레이싱카들에 대한 기능 (Cars)
    * 게임에 참가하는 레이싱카들의 리스트이다.
    * (,) 를 기준으로 레이싱카 이름을 입력받아 생성할 수 있다.
    
* 레이싱카 한개에 대한 이동결과 (CarMoveHistory)
    * 해당 레이싱카의 이름과 현재 위치에 대한 정보를 가진다.
    
* 한번의 움직임에 대한 이동결과들 (CarMoveHistories)
    * 레이싱카 한개에 대한 이동결과들의 리스트이다.
    * 가장 앞선 레이싱카의 이름과 위치를 가져올 수 있다.

* 메인 게임
    * 게임을 시작한다.
    * 우승자를 선정한다.
    * 매 라운드에 CarMoveHistories 를 저장한다. (출력 및 우승자 선정에 사용)

* 입출력에 대한 기능
    * 사용자는 게임에 참여할 레이싱카들의 이름을 입력한다.
    * 레이싱카가 몇번 움질일지 값을 입력한다.
    * 게임을 완료한 후 누가 우승했는지 알려준다. 우승자는 한 명 이상일 수 있다.
