# 자동차 경주 게임
## 진행 방법
* 자동차 경주 게임 요구사항을 파악한다.
* 요구사항에 대한 구현을 완료한 후 자신의 github 아이디에 해당하는 브랜치에 Pull Request(이하 PR)를 통해 과제를 제출한다.

## 과제 제출 과정
* [과제 제출 방법](https://github.com/next-step/nextstep-docs/tree/master/precourse)

## 구현 기능 정리
* 레이스 컨트롤러 클래스
  * 레이스 서비스 클래스를 호출하여야한다.
    * 참여자와 시도 회수를 입력 받아 전달한다.
  * 레이스 종료 후 결과물을 반환 받아 출력한다.
* 레이스 서비스 클래스
    * 시도 회수만큼 레이스를 진행시켜야한다.
    * 레이스가 종료되면 레이스 결과를 반환하여야한다.
* 레이스 게임 클래스(게임의 정보)
    * 참여한 자동차들의 목록을 가지고 있어야한다.
    * 참여한 자동차들을 이동 혹은 정지 시켜야한다.
      * `0 ~ 9의 난수를 이용하여 4 이상일 경우 전진, 3 이하는 정지.`
    * 레이스의 내역을 가지고 있어야한다.
    * 최종 완료시 우승자를 반환하여야 한다.
        * `우승자가 한명 이상일 경우, 쉼표(,)로 이름을 구분하여 반환하여야 한다.`
* 자동차 클래스
  * 자동차 이름을 가지고 있어야한다.
  * 전진 한 회수를 가지고 있어야한다.
* 라운드 클래스
  * 라운드별 레이스 내역을 보관한다.
* 입력 클래스
  * 레이스에 참여할 자동차 이름을 입력 받아야한다.
      * `이름의 길이는 5자 이하로 충족해야한다.(0의 길이의 이름은 예외처리 예정) `
  * 레이스 시도 회수를 입력 받아야한다.
      * `1회 이상 시도를 충족해야한다.`
* 출력 클래스
  * 오류 발생시 `[ERROR]` 문자열을 추가하여 출력한다.
