# 자동차 경주 게임
## 학습 목표 (주관적인)
* TDD 학습
  * 테스트 케이스를 작성 후 실패, 성공, 리펙토링 과정을 반복하며 구현체를 작성한다.
  * 테스트 케이스를 통과할 수 있는 최소한의 코드로 구현 (예측하여 over engineering 하지말 것)
* Domain 기반으로 설계
* 일급콜렉션 활용
* 원시값과 문자열 Wrapping 활용
* 구현 기능 단위 commit
  * 기능 단위로 test가 완료된 경우 commit 한다.
  * 한 가지 기능이 아닌 docs, feat, test, refactor, fix 등이 뒤썩여 commit 되지 않도록 한다. 

## 요구사항
* 주어진 횟수 동안 n대의 자동차는 전진 또는 멈출 수 있다.
* 각 자동차에 이름을 부여할 수 있다. 전진하는 자동차를 출력할 때 자동차 이름을 같이 출력한다.
* 자동차 이름은 쉼표(,)를 기준으로 구분하며 이름은 5자이하만 가능하다.
* 사용자는 몇번의 이동을 할 것인지를 입력할 수 있어야 한다.
* 전진하는 조건은 0에서 9사이에서 random 값을 구한 후 random 값이 4이상 일 경우 전진하고, 3이하의 값이면 멈춘다.
* 자동차 경주 게임을 완료한 후 누가 우승했는지를 알려준다.우승자는 한명이상 일 수 있다.

## 기능목록
* 자동차 이름 입력
  * 쉼표(,)를 기준으로 구분
  * 이름은 5자 이하
  * 이름 중복 체크
* 자동차 이동 횟수 입력
  * 0보다 큰 숫자
* 실행 출력 (횟수별)
  * 자동차 이름 표기
  * 자동차 전진 횟수 표기 (ex: apple:---)
  * 0~9 사이 random 값이 4이상이면 전진, 아닐 경우 멈춤
* 결과 출력
  * 가장 많이 이동한 자동차들을 모두 출력

## 구현목록

### 테스트 케이스
* RandomNumber 생성
```text
0~9 사이 random
```

* CarName 생성
```text  
5자리 이하 string을 wrapping
사용자에게 입력 받는 값으로 유효성이 올바르지 않을 경우 오류 발생
최초 생성 후 읨이 변경 불가
```

* CarPosition 생성
```text  
0부터 시작하는 int를 wrapping
최초 생성 후 읨이 변경 불가
자동차이동 명령이 있을 경우 +1 
```

* Car 생성
```text
자동차이름, 자동차위치를 생성 후 자동차에 할당
자동차목록에서 중복 여부를 확인하기 위해 equals, hashCode method 구현

name
페라리 -> 생성
에스턴마틴 -> 생성
레이싱포인트 -> 오류
```

* Car 이동
```text
자동차 전진, 멈춤 상태에 대한 enum 생성
enum에 랜덤숫자를 메세지로 전달하면 조건에 따라 전진 또는 멈춤 상태를 반환
```

* Cars 생성
```text
사용자에게 자동차들의 이름을 ,를 구분자로 입력 받아 생성 후 목록 저장
자동차 이름이 유효하지 않은 경우 또는 자동차 이름이 중복되는 경우 오류 발생

user input: 하스,페라리,르노,하스,현대
Collection의 contains를 이용해 자동차가 중복되는지 체크 후 중복 시 오류 발생
```

* Cars 이동
```text
Cars에 저장된 Car들이 RandomNumber를 통해 MoveStatus(전진, 멈춤)를 Car에 전달하여
전진 상태일 경우 CarPosition을 증가
```

* Cars 우승자 목록
```text
사용자 입력 횟수대로 Cars 이동을 완료한 후
Cars에 포함된 Car의 position을 비교하여 가장 값이 큰 position과 같이 같은 Car들을 반환
```
