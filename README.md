# 자동차 경주 게임
## 기능 요구사항 요약
    * 주어진 횟수동안 전진하는 자동차를 생성하고,
    * 전진하는 조건에 맞추어서 전진한 후, 
    * 우승자를 알려주는 프로그램을 구현한다.

***

## 프로그래밍 요구사항
    * 일급 콜렉션을 활용해 구현한다.
    * 모든 원시값과 문자열을 포장한다.
***
## 구현 기능 목록

### 1. 메인 클래스
    * 프로그램을 구동하기 위한 메인 클래스 역할
    * 시작점이라는 것을 명시하기 위해서 메인클래스로 정의, 게임을 시작하기만 함. 

### 2. 게임 클래스
    * 정의 : com.tjnam.racingcargame.RacingGame
    * 정의 : 레이싱 게임의 진행과 관련한 클래스
    * 게임 전체의 플로우를 관리 및 구현하는 클래스
    * UI입력에 대해서 출력하는 것을 담당한다 -> 게임 진행과 연관

#### 2.1. 레이싱 횟수 입력 클래스
    * 사용자(게이머)에게 레이싱 횟수를 입력받는다. 
    * 입력받은 횟수를 바탕으로 레이싱 게임을 수행할 횟수를 정의한다.

#### 2.2. 랜덤값 생성 클래스
    * 자동차 별 전진 여부를 판단하기 위한 랜덤값을 생성하는 클래스

#### 2.3. 이동여부 결정 판단 클래스 
    * 이동여부를 판단하고 이동을 수행하는 클래스
***
### 3. 자동차들 클래스 (com.tjnam.racingcargame.Cars)
    * 레이싱 게임에 참여하기 위한 자동차 클래스
    * 이름을 받아서 생성한다.
    * 자동차의 컬렉션 클래스
    * 승자를 결정하기 위한 로직을 포함한다. 

#### 3.1. 자동차 클래스
    * 하나의 자동차를 나타내는 클래스 
    * 자동차 이름으로 생성되는 final 클래스
    * 움직일 수 있다. 

#### 3.2. 자동차 이름 클래스
    * 다섯글자의 자동차 이름을 정의한다.

#### 3.2. 자동차 이름입력 클래스
    * 사용자의 자동차 이름 입력을 받는다. 
    * 입력을 기반으로 자동차들을 생성한다
    * 사용자 입력 클래스로 정의하는 것도 나쁘지 않을 듯 (TBD)

#### 3.2.1. 자동차 이름입력 Validator 클래스 
    * Validation을 어디에서 할 지에 대한 고민. 
    * 입력과 Validation을 동시에 하는게 좋을거 같은데, 분리해도 좋을거 같기도 함. 

#### 3.3. 자동차 위치 클래스
    * 현재의 자동차 위치를 나타내는 클래스
