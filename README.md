
## 자동차 경주 게임

###1. 게임 설명
   - 자동차를 입력한 횟수만큼 랜덤으로 이동시켜 가장 앞선 자동차를 출력하는 게임이다.

<br>

###2. 게임 규칙
   - 주어진 횟수 동안 n대의 자동차는 전진 또는 멈출 수 있다.  
   - 각 자동차에 이름을 부여할 수 있다. 전진하는 자동차를 출력할 때 자동차 이름을 같이 출력한다.  
   - 자동차 이름은 쉼표(,)를 기준으로 구분하며 이름은 5자 이하만 가능하다.  
   - 사용자는 몇 번의 이동을 할 것인지를 입력할 수 있어야 한다.  
   - 전진하는 조건은 0에서 9사이에서 random값을 구한 후   
     random값이 4이상일 경우 전진하고, 3이하의 값이면 멈춘다.  
   - 자동차 경주 게임을 완료한 후 누가 우승했는지를 알려준다. 우승자는 한 명 이상일 수 있다.  

<br>

###3. 실행 방법
   - 별도의 서버를 구축하지 않았으므로 IDE 내에서 Application.main을 Run하여 테스트 가능.

<br>

###4. 실행 예시
    경주할 자동차 이름을 입력하세요. (이름은쉼표(,)기준으로구분)
    pobi,crong,honux
    시도할 회수는 몇회인가요?
    5
    
    실행결과
    pobi:-
    crong:-
    honux:-
    
    pobi:--
    crong:-
    honux:--
    
    프로그램 실행 결과
    pobi:---
    crong:--
    honux:---
    
    pobi:----
    crong:---
    honux:----
    
    pobi:-----
    crong:----
    honux:-----
    
    pobi,honux가 최종 우승했습니다
    
    

<br>

###5. 기능 구현 리스트
- [ ] 자동차 이름 입력요청 문구 출력 (view)
    - [ ] 쉼표 기준으로 동적으로 입력받음 (view/input)
    - [ ] 이름은 5글자 이하 (valid)

- [ ] 시도할 횟수 입력요청 문구 출력 (view)
    - [ ] 시도할 횟수를 입력받음 (view/input)
    - [ ] 숫자만 입력받음 (valid)

- [X] 0~9 사이의 랜덤값을 생성 (feat)
    - [X] 숫자 유효성 체크 (feat)
    - [X] 4 이상일 경우 go, 아닐경우 stop (feat)

- [ ] 게임 완료 (feat)
    - [ ] 우승한 차 판별 (feat)
    - [ ] 우승한 차 이름 출력 (view)

<br>

###6. 개발 순서 (feedback 반영)  
  view가 있는 부분은 처음부터 TDD로 만들어나가기 어려우므로 기능 위주로 시작해본다.  
  따라서 3번 기능부터 TDD 진행한다.  

  - 클래스는 어떻게 설계하는게 좋을까?  
    1주차 동영상에서 보면 야구게임에서 Ball 클래스를 만들어 position과 number를 입력받았던데,   
    '나도 이렇게 할걸', '클래스 설계가 코딩을 쉽게 하는구나.' 이라는 생각이 들었음.  

    자동차의 '이름'과 '위치값'을 저장할 클래스,  
    자동차가 1회차 실행시마다 갖게 될 1~9 사이의 숫자를 가질 클래스 부터 시작하여,  
    + 게임 플레이  
    + 난수 생성  
    + view  
    + 메인  
    까지 TDD로 만들어 볼 예정.

<br>

###7. 개발 방법 (feedback 반영)

1주차 피드백 동영상에서 배운대로 어떻게 개발해볼 것인가?  

(1) TDD (TFD)  
  - 작은 단위로 쪼개어 테스트코드를 작성하면서 거꾸로 method를 만들어간다.  
    도메인, 객체 설계 능력을 길러간다.  
    단순히 개발하면서 단위테스트하는 것이 TDD라기 보단,  
    테스트를 통해 메서드를 완성해가는 것..  
    
(2) 일급 콜렉션  
  - 콜렉션을 포함한 클래스는 반드시 다른 멤버 변수가 없어야 한다.  

(3) 원시값 포장
  - 하나하나 validation을 만들기보다는,  
    클래스 생성자에서 제어하여, 원시값 값 자체를 포장한다.  
  -> 일급 콜렉션을 활용하여, 원시값을 포장하는 연습을 해보면 될 듯 하다.  

(4) 리팩토링
  - 상수, enum을 활용.  
  - 생성자는 간단하게 줄일 수 있음. (intelliJ method 추출)  
  - 객체에 메세지를 던지자.
