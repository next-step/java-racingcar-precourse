# 🚗 자동차 경주 게임 🚗
## 📌 진행 방법
* 숫자 야구 게임 요구사항을 파악한다.
* 요구사항에 대한 구현을 완료한 후 자신의 github 아이디에 해당하는 브랜치에 Pull Request(이하 PR)를 통해 과제를 제출한다.

## 📌 과제 제출 과정
* [과제 제출 방법](https://github.com/next-step/nextstep-docs/tree/master/precourse)







## 🍿 요구 사항

* 주어진 횟수 동안 n 대의 자동차는 전진 또는 멈출 수 있다.
* 각 자동차에 이름을 부여할 수 있다. 전진하는 자동차를 출력할 때 자동차의 이름을 같이 출력한다.
* 자동차의 이름은 쉼표","를 기준으로 구분하며 이름은 5자 이하만 가능한다.
* 사용자는 몇 번의 이동을 할 것인지를 입력할 수 있어야한다.
* 전진하는 조건은 0에서 9 사에서 random 값을 구한 후 random 값이 4 이상일 경우 전진하고, 3 이하의 값이면 멈춘다.
* 자동차 경주게임을 완료한 후 누가 우승했는지 알려준다. 우승자는 한명 이상일 수 있다.



## ❗ 규칙 추가

* 자동차 객체 이름에 공백을 허용하지 않는다



## 🤔 기능 정의



### 🍉 Step 1 : 숫자 생성 기능

1. 0 - 9 : 숫자 구함 
   * 숫자 생성이 0 -9 까지인가?
   * 숫자 범위 입력시 예외처리 확인



### 🍉 Step 2 : 자동차 객체

1. 자동차 이름 

   * 자동차 이름이 5자리 이하 인가? 

   * 이름에 공백이 들어가 있는가?

     

2. 자동차 위치 

   * 랜덤 숫자 객체에서 들어오는 숫자가 올바른 범위인가?

   * 더 이상 움직일 수 없는 범위인가? (int 범위 초과)

   * 4 이상의 숫자가 들어올 경우 전진, 3이하는 stay

     * 4이상일때 전진하는가?

     * 3이하일 경우 멈춰있는가?

       

3. 자동차 움직임

   * 4턴동안 3번 이동하고 1번 멈췄을 때 결과가 일치하나
   * 4턴 동안 아예 멈춰있을 경우 결과가 일치하나
   * 4턴동안 4번 다 이동할 경우 결과가 일치하나?





### 🍉 Step 3 :  게임 진행자 기능

1. 몇번을 이동 할지 입력

   * 숫자 아닌 값이 들어오면? 

   * int 의 범위보다 큰 값이 들어오면?

     





### 🍉 Step 4 : 자동차 리스트 객체

1. 자동차 이름을 받을때
   * 자동차 이름이 동일 한 경우 인풋 예외 처리

2. 자동차 객체 비교 기능
   * 우승자가 한명일 경우
   * 우승자가 여러명일 경우
