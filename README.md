# 자동차 경주 게임
## 진행 방법
* 숫자 야구 게임 요구사항을 파악한다.
* 요구사항에 대한 구현을 완료한 후 자신의 github 아이디에 해당하는 브랜치에 Pull Request(이하 PR)를 통해 과제를 제출한다.

## 과제 제출 과정
* [과제 제출 방법](https://github.com/next-step/nextstep-docs/tree/master/precourse)


## 요구사항
- 자동차는 이름을 가진다.[v]
  - 자동차 이름은 쉼표를 기준으로 구분하며, 이름은 1자 이상, 5자 이하만 가능하다.[v]
  - 자동차 이름은 널값이나 빈 값을 가질 수 없다[v]
- 어플리케이션이 시작하면 자동차 이름을 입력받는다.
  - 어플리케이션 시작시 "경주할 자동차 이름을 입력하세요. (이름은 쉼표(,) 기준으로 구분)" 라는 메시지를 출력한 후 사용자에게 입력받는다.
  - 입력 받은 자동차 이름에 문제가 있다면, "{문제 내용} 다시 입력하세요" 라는 메시지를 출력한다.
- 입력 받은 자동차 이름에 문제가 있다면, 문제 내용을 담은 오류를 발생시킨다.[v]
  - 자동차 이름이 10개 이상이라면, "자동차 이름은 10개까지 입력 가능합니다" 라는 메시지를 담은 오류를 발생시킨다.[v]
  - 자동차 이름이 없다면, "입력된 차량 대수가 0대입니다. 다시 입력하세요" 라는 메시지를 담은 오류를 발생시킨다.[v]
  - 자동차 이름이 한 개라면, 경주가 불가능하므로 "입력된 차량 대수가 1대입니다. 다시 입력하세요" 라는 메시지를 담은 오류를 발생시킨다.[v]  
- 자동차 이름의 입력이 끝난 후, 사용자는 몇 번의 이동을 할 것인지 입력한다.
    - "시도할회수는몇회인가요?" 라는 메시지 출력 후 이동 횟수를 입력을 받는다.
    - 이동 횟수는 1 이상, 20 이하로 제한한다
- 사용자가 입력한 이동 횟수에 따라 이동을 한다.[v]
    - 매 이동 횟수마다, "{각 차량 이름} : {이동한 횟수만큼 -}"를 차량마다 출력한다.
- 차량의 이동은 랜덤값으로 0~9를 발생시키고, 랜덤값이 4이상이면 전진, 3이하의 값이면 멈춘다. [v]  
  - 차량은 입력받은 값이 4 이상이면 이동하고, 3 이하면 이동하지 않는다.[v]
  - 트랙정보에서 차량에 각각 랜덤값을 전달한다.[v]
- 누가 우승했는지를 계산한다[v]
  - 여러 차량이 공동으로 우승하였을 경우, 우승 차량 리스트를 리턴한다.[v]
- 누가 우승했는지를 출력한다.
  - "{우승한 차량 이름}가 최종우승 했습니다."를 출력한 후 어플리케이션을 종료한다.
- 차량의 이동 횟수가 가장 높은 차량이 우승 차량이다.
  - 우승한 차량은 1대 이상이다.
