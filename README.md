# 레이싱 게임

## 개요

레이싱 게임은 정해진 횟수 안에 가장 멀리 이동한 레이싱카를 우승으로 결정하는 게임이다.

## 이력

| 버전 | 업데이트 내역 |
| --- | --- |
| v.1 | 레이싱카 용어사전, 요구사항, 모델링 작성 |
| v.2 | 주사위 요구사항 추가 |
| v.3 | 주사위 요구사항 변경<br>\- 주사위는 값만 반환하고\, 주행 가능여부를 판단하지 않는다\.<br><br>규칙 추가<br>\- 주사위의 값을 보고\, 레이싱카가 주행 가능한지 알려준다\. |
| v.4 | 서킷 모델링, 요구사항 추가 |

## 용어사전

| 한글명 | 영문명 | 설명 |
| --- | --- | --- |
| 주사위 | Dice | 주행 가능 여부를 결정하는 도구 |
| 레이싱 | Racing | 주사위의 숫자에 상태를 결정한다. |
| 주행 | Forward | 레이싱카는 주행한다. |
| 정지 | Stop | 레이싱카는 주행하지 않는다. |
| 서킷 | Circuit | 레이싱카가 주행하는 트랙 |
| 랩 | Lap | 서킷을 완주해야하는 횟수 |
| 레이싱카 | RacingCar | 경기에 참여한 레이싱카 |
| 랩결과 | LapResult | 레이싱카의 진행상황을 기록한다. |
| 최종결과 | FinalResult | 랩결과를 누적 기록한다. 최종 우승자를 기록한다. |

## 요구사항

### 레이싱카
* 레이싱카의 이름은 규칙에 따라 설정한다.
  * 이름은 1글자 이상, 5글자 이하로 정한다.
* 레이싱카는 `주사위`를 던져 규칙에 따라 주행/정지를 반복한다.
  * 주사위 값이 0이상 3이하인 경우 정지하고, 4이상 9이하인 경우 주행한다.

### 주사위
* 주사위는 규칙에 정의한 최솟값, 최댓값 사이의 값을 하나 반환한다.

### 규칙
* `레이싱카` 의 이름은 1글자 이상, 5글자 이하이다.
* `주사위`의 눈은 0이상, 9이하이다.
* `주사위`의 눈이 0이상 3이하일 경우, `레이싱카`는 `정지` 하고, 4이상 9이하일 경우 `레이싱카`는 `주행` 한다.
* `레이싱카` 가 `주행` 할 경우, 한 칸 이동한다.
* 가장 많이 이동한 `레이싱카` 가 우승이다. 우승이 2명이상인 경우 공동 우승으로 한다.

### 서킷
- 입력한 `랩` 수 만큼, 경기를 진행한다.
- `랩` 마다 레이싱카의 `랩기록`을 기록한다.
- 경기를 마치고, `최종결과`를 작성하고, `규칙`에 따라 우승자를 결정한다.

## 모델링

### 레이싱카

* `레이싱카`는 `이름`, `현재위치`, `주사위` 를 가진다.
* `레이싱카`는 `현재위치` 를 기록한다.
* `레이싱카`는 `주사위`를 던져 `주행`/`정지` 을 결정한다.

### 규칙

* `레이싱카`의 이름 제약사항을 정의한다.
* `레이싱카` 의 `주행`/`정지` 상태를 정의한다
* `주사위` 눈금의 최솟값/최댓값을 정의한다.
* `우승` 조건을 정의한다

### 서킷
- 대회에 참여한 `레이싱카`목록과 `랩` 수를 갖는다.
- 매 `랩`마다 `랩결과`를 작성한다.
- 경기가 끝난 후, `최종결과`를 작성한다.
