# 자동차 경주 게임
## 진행 방법
* 자동차 경주 게임 요구사항을 파악한다.
* 요구사항에 대한 구현을 완료한 후 자신의 github 아이디에 해당하는 브랜치에 Pull Request(이하 PR)를 통해 과제를 제출한다.

## 과제 제출 과정
* [과제 제출 방법](https://github.com/next-step/nextstep-docs/tree/master/precourse)

## 리마인드
* 일급콜랙션 활용
* 모든 원시값과 문자열 포장 

## 기능목록

* RACE_001 : 자동차 이름 문자열
  1. 입력받은 값 검증
  

* RACE_002 : 자동차 이름(자동차 이름은 쉼표를 기준으로 구분하며 이름은 5글자 이하)
  1. 자동차 이름 검증
  
  
* RACE_003 : 총 진행 바퀴 수
  1. 입력받은 값 검증
  

* RACE_004 : 주행거리
  1. 주행거리 계산
  

* RACE_005 : 주행거리 표시
  1. 주행거리 표시
  

* RACE_006 : 자동차
  1. 전진 4 이상일 경우
  
  
* RACE_007 : 자동차 리스트 불변 리스트 생성(일급 콜랙션 비즈니스 로직 집중화)
  1. 입력받은 레이싱 회수만큼 레이싱 실행
  2. 우승자 반환
  

* RACE_008 : 자동차 공장
  1. 입력받은 쉼표를 기준으로 자동차 객체 생성
  2. 불변 리스트 주입


* RACE_009 : 최고 기록
  1. 최고 기록 추출 로직 구성
  2. 최고 기록 반환
  

* RACE_010 : 우승자
  1. 최고 기록 기반 우승자 선발 로직 구성
  2. 우승자 계산 및 반환
  
  
* RACE_011 : Official 경기 진행자
  1. 경기 진행 안내 
  2. 자동차 이름 입력 받기
  3. 레이싱 총 진행 바퀴수 입력 받기
  4. 결과 공표
  
    
* RACE_012 : 레이싱 컨트롤러
  1. 진행자 생성
  2. 자동차 공장 생성
      1. 오류 발생시 다시 진행
  2. 레이스 진행
      1. 오류 발생시 다시 진행
  

* 부하테스트
  1. TotalLaps 매우 큰값 입력시 지연 발생
  2. MAX TotalLaps Validation 추가 검토
  

* 리팩토링
  1. GUIDE 분리 검토
