# 자동차 경주 게임
## 진행 방법
* 자동차 경주 게임 요구사항을 파악한다.
* 요구사항에 대한 구현을 완료한 후 자신의 github 아이디에 해당하는 브랜치에 Pull Request(이하 PR)를 통해 과제를 제출한다.

## 과제 제출 과정
* [과제 제출 방법](https://github.com/next-step/nextstep-docs/tree/master/precourse)



## 기능 요구사항

- 주어진 횟수 동안 n대의 자동차는 전진 또는 멈출 수 있다.
  - [ ] ``Car`` 객체들을 담고 있는 ``일급 컬렉션 Cars``를 생성한다.
  - [ ] ``Cars`` 객체는 사용자가 입력한 이동 횟수(n) 만큼  메소드를 호출한다.
  - [ ] ``Cars`` 객체가 갖고 있는 ``Car 컬렉션``의 객체들에게 전진할 것인지, 멈출 것인지 질의한다.
- 각 자동차에 이름을 부여할 수 있다. 전진하는 자동차를 출력할 때 이름을 같이 출력한다.
  - [ ] ``Car`` 객체는 ``이름 (Name)`` 과 ``위치 (Position)`` 객체 멤버를 갖는다.
  - [ ] ``이름 (Name)``과 ``위치 (Position)`` 객체는 ``모든 원시값과 문자열을 포장한다.``라는 요구사항을 반영한다.
- 자동차 경주 게임을 완료한 후 누가 우승했는지를 알려준다.
  - [ ] ``Cars`` 객체에게 우승자가 누구인지 질의하고 우승자 리스트를 리턴받는다.

**세부 기능 요구사항은 테스트 사항 참고**



## 테스트 사항

#### Position 객체

- ~~외부에서 주입받은 값을 판단하여 ``위치``를 전진할 것인지, 멈출 것인지 판단한다.~~
- ~~외부에서 주입받은 값이 음수일 경우 예외를 발생시킨다.~~
- 외부에서 주입받은 값을 판단하는 로직을 외부 객체에 위임한다. 즉, ``Generator 인터페이스``를 사용하여 해당 구현체의 ``canMove`` 메소드에게 이동이 가능한지 질의하고 ``true``이면 전진, ``false`` 경우 정지한다.

#### NumberGenerator 인터페이스

- ``Random`` 클래스를 사용하여 자동차가 전진할 수 있는지 여부를 리턴한다.
- `Random` 클래스가 생성한 값이 `기준값` 이상이면 `true`, 그렇지 않으면 `false`를 리턴한다.



#### Name 객체

- 이름은 5자 이하만 가능하며, 5자를 초과할 경우 예외를 발생시킨다.

#### Car 객체

- ``Position`` 객체에게 전진할 것인지 멈출 것인지 질의한다.



## 주의사항

- ``객체``로부터 값을 얻어 로직을 수행하지 않고, ``객체``에게 로직을 수행을 **요청**한다.
