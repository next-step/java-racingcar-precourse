# 레이싱 게임

## 개요

레이싱 게임은 정해진 횟수 안에 가장 멀리 이동한 레이싱카를 우승으로 결정하는 게임이다.

| 버전 | 업데이트 내역 |
| --- | --- |
| v.1 | 레이싱카 용어사전, 요구사항, 모델링 작성 |
| v.2 | 주사위 요구사항 추가 |
| v.3 | 주사위 요구사항 변경<br>\- 주사위는 값만 반환하고\, 주행 가능여부를 판단하지 않는다\.<br><br>규칙 추가<br>\- 주사위의 값을 보고\, 레이싱카가 주행 가능한지 알려준다\. |
| v.4 | 서킷 모델링, 요구사항 추가 |
| v.5 | 레이싱카 서킷에서 주사위를 던지고, 레이싱카는 결과만 반영해 진행하도록 수정<br><br>규칙 분리<br>\- 주사위 판별\, 우승자 판별 |

## 요구사항

### 서킷

* 레이싱에는 레이싱카는 1대 이상 참여해야 한다.
* 입력한 `랩` 수 만큼, 경기를 진행한다.
* `레이싱카` 마다 주사위를 던지고, 이동규칙 결과값을 레이싱카에게 전달한다.
* `랩` 을 마치면, 참여한 모든 레이싱카의 `랩기록` 기록한다.

### 레이싱카

* 레이싱카의 이름은 1글자 이상 5글자 이하로 정한다.
  * 이름은 공백을 포함하지 않는다.
* 레이싱카의 위치는 0 부터 시작한다.
* `주사위` 값으로 결정된 `이동` 에 설정한 값 만큼 위치를 변경한다.
* 위치를 이동한다음 랩기록을 반환한다.


### 랩

* 랩 을 설정한다.
  * 랩은 1이상이어야 한다.


### 주사위

* 주사위는 규칙에 정의한 최솟값, 최댓값 사이의 임의의 값을 생성한다.
  * `10면체주사위` 는 0에서 9사이의 임의의 값을 생성한다.

### 이동규칙

* 주사위의 값이 `0` 이상, `3` 이하이면 정지(STOP) 를 반환한다.
* 주사위의 값이 `4` 이상, `9` 이하이면 주행(FORWARD) 을 반환한다.

### 우승자결정규칙

* `최종기록` 를 기반으로 우승자를 결정한다.
* `가장멀리이동하면우승 규칙` 을 적용한다.
* 가장 멀리 이동한 레이싱카가 2대 이상이면, 공동 우승으로 정한다.

### 출력

* 랩 별 `레이싱카`의 진행 상황을 출력한다.
  * 진행상황은 `-` 으로 표시한다.


## 모델링

### 레이싱 게임
- `RacingGame` 은 `서킷`을 가진다.

### 레이싱카
* `레이싱카` 는 `이름`, `현재위치` 를 가진다
* `레이싱카`는 `현재위치` 를 기록한다.
* `RacingCars`에 속한다.

### 서킷
* `서킷` 은 `주사위`, `이동규칙` 을 가진다
* `이동규칙` 은 `주행`, `정지` 상태를 갖는다.
* `주사위`와 `이동규칙` 을 이용해 경기에 참여한 `레이싱카`의 움직임을 결정한다.
* `랩` 마다 모든 `레이싱카`의 기록을 기록한다.
* `RacingCars` 를 `Lap` 만큼 반복 실행한다.

### 최종기록
- `최종기록` 은 랩별 `RacingCars` 의 기록을 가진다.
- `최종기록` 은 누적기록이다.
- `최종기록` 의 가장 마지막 기록은 우승자 결정에 사용한다.

### 우승자 결정
- `우승자` 은 `최종기록`을 `우승자결정규칙`에 맞게 해석해 결정한다.
