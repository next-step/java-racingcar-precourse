# Racing car

## 기능 목록

1. 게임 설정을 한다.
   1. 자동차 이름을 입력으로 받는다.
   2. 라운드 횟수를 입력으로 받는다.
2. 라운드를 진행한다.
   1. 라운드 진행에 따라 자동차 위치를 업데이트한다.
   2. 라운드 종료시 마다 현재 자동차 위치를 화면에 업데이트한다.
3. 결과를 출력한다.
   1. 우승자를 출력한다.

## 객체 역할과 책임

- RacingGameController
  - RoundOrganizer에게 자동차 생성을 위한 메시지를 보낸다.
  - 라운드 회수를 입력으로 받는다.
      - 라운드 횟수 입력을 받는다.
      - 잘못된 입력이 들어온 경우 예외를 던진다.
  - 각 라운드를 진행한다.
  - 참여한 자동차를 관리한다.
- RacingOrganizer
  - 참여하는 자동차를 생성한다.
    - 5 글자 이상인 입력을 받을 수 없다.
    - 잘못된 입력이 들어온 경우 예외를 던진다.
- Car
  - Engine(Random) 결과를 통해 이동할지 결정한다.
  - 이름과 위치 정보를 관리한다.
- QuestionView
  - 게임 설정에 필요한 질문 뷰
- ErrorView
  - 잘못된 입력에 필요한 응답 뷰
- GameBoardView
  - 현재 자동차 위치를 전시한다.
  - 우승자를 출력한다.

## 고민 사항

- 이름이 같은 자동차는 식별이 불가능 하다고 생각해 자체적으로 새로운 제약 조건을 만들었습니다.
새로운 요구사항을 만족하기 위해 자동차 객체를 Set 자료구조로 관리하려고 했지만,
그런 경우 프로그램 결과가 자동차 출발 순서에 의존성이 생겨 테스트가 어려워집니다.
이를 위해 SortedSet 자료구조와 자동차 객체가 comparable 인터페이스를 구현해 해결할 수 있지만,
오버엔지니어링이라고 판단되며 이름이 같아도 순서를 통해 자동차가 식별 가능하기 때문에 중복 이름 제약 조건을 제거합니다.
