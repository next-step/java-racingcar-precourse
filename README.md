# 자동차 경주 게임
## 진행 방법
* 자동차 경주 게임 요구사항을 파악한다.
* 요구사항에 대한 구현을 완료한 후 자신의 github 아이디에 해당하는 브랜치에 Pull Request(이하 PR)를 통해 과제를 제출한다.

## 과제 제출 과정
* [과제 제출 방법](https://github.com/next-step/nextstep-docs/tree/master/precourse)

## 기능 요구사항
- [ ] 주어진 횟수 동안 n대의 자동차는 전진 또는 멈출 수 있다.
- [ ] 각 자동차에 이름을 부여할 수 있다.
- [ ] 전진하는 자동차를 출력할 때 자동차 이름을 같이 출력한다.
- [ ] 자동차 이름은 쉼표(,) 를 기준으로 구분하며 이름은 5자 이하만 가능하다.
- [ ] 사용자는 몇 번의 이동을 할 것인지를 입력할 수 있어야 한다.
- [ ] 전진하는 조건은 0에서 9사이에서 random 값을 구한 후, 4 이상인 경우 전진, 3 이하인 경우 멈춘다.
- [ ] 자동차 경주 게임을 완료한 후 누가 우승했는지를 알려준다
- [ ] 우승자가 한 명 이상일 경우, 쉼표(,) 로 이름을 구분해 출력한다.
- [ ] 사용자가 잘못된 값을 입력할 경우 "[ERROR]" 로 시작하는 에러 메시지를 출력 후 입력을 다시 받는다.

## 기능 목록
### 자동차
#### 자동차 도메인
- [x] 자동차에 이름, 위치를 부여하여 생성한다.
- [x] 자동차 움직임 전략에 따라 자동차를 움직인다.
- [x] 자동차의 이름과 현재 위치를 출력한다.
- [x] 임의의 위치와 자동차의 위치를 비교하여 더 큰 위치를 반환한다.

#### 자동차 위치 도메인

- [x] 0보다 큰 위치만을 갖는다.
- [x] 위치를 증가시킨다.
- [x] 자동차의 위치를 출력한다. (0 이면 '', 3 이면'---', 4면 '----')

#### 자동차 이름 도메인

- [x] 1보다 크거나 같으며, 5보다 작거나 같은 길이의 이름만을 갖는다.
- [x] 자동차의 이름을 출력한다.

#### 여러개 자동차들의 도메인

- [x] 자동차들의 위치 및 이름을 반환한다.
- [x] 자동차들의 위치 중 가장 큰 위치를 반환한다.
- [x] 자동차들을 움직인다.

#### 승리한 자동차들의 도메인

- [x] 승리한 자동차들의 이름을 쉼표로 구분해 출력한다.
- [x] 승리한 자동차들을 가져온다.

### 시도횟수 도메인

- [x] 0보다 크거나 같은 시도횟수만을 갖는다.
- [x] 시도횟수를 감소시킨다.

### 랜덤

#### 숫자 생성 도메인

- [x] 숫자를 생성한다.

#### 랜덤 숫자 생성 도메인

- [x] 0-9 사이의 값을 반환한다.
- [x] 랜덤 숫자 생성 인스턴스를 반환한다. (싱글턴)

### 움직임

#### 움직임 Strategy

- [x] 4 이상의 값이 생성되는 경우, 움직인다.

#### 랜덤 움직임 Strategy

- [x] 랜덤한 값을 생성하고, 4 이상의 값이 생성되는 경우, 움직인다.
- [x] 랜덤 움직임 Strategy 인스턴스를 반환한다. (싱글턴)

### View

#### 입력받는 View

- [x] 콤마로 구분된 경주할 자동차의 이름을 입력받아 List<String> 으로 반환한다.
- [x] 시도할 횟수를 입력받고, Integer 로 반환한다.
  - [x] 숫자형태 문자열이 아니면 예외를 던진다.

#### 출력하는 View

- [x] [ERROR] 로 시작하는 메시지를 출력한다.
- [x] '경주할 자동차 이름을 입력하세요.(이름은 쉼표(,) 기준으로 구분)' 을 출력한다.
- [x] '시도할 회수는 몇회인가요?' 를 출력한다.
- [x] '실행결과' 를 출력한다.
- [x] 중간중간의 자동차들의 움직이는 상태를 출력한다.
- [x] '최종 우승자는 ???,??? 입니다.' 를 출력한다.

### 레이싱게임 컨트롤러

- [x] 자동차들 객체 (RacingCars) 를 생성한다.
  - [x] 입력받는 View 를 통해 자동차의 이름을 입력받는다.
  - [x] 자동차 객체를 생성한다.
  - [x] 자동차들 객체를 생성한다.
  - [x] 자동차 이름이 6자 이상인경우 에러를 출력한다.
- [ ] 입력받는 View 를 통해 시도횟수를 입력받는다.
  - [ ] 자동차 위치 객체를 이름만큼 생성한다.
- [ ] 위의 데이터를 기반으로 자동차 객체를 생성한다.
- [ ] 여러개 자동차 객체를 자동차들로 감싸서 생성한다.
- [ ] 게임을 시작한다.
  - [ ] 반복적으로 자동차들을 움직인다.
    - [ ] 랜덤도메인에서 자동차 대수만큼 random 값을 받아온다.
  - [ ] 자동차들의 위치를 출력한다.
  - [ ] 반복적으로 게임의 종료여부를 판단한다.
- [ ] 게임을 종료한다.
  - [ ] 우승 자동차를 출력한다.
