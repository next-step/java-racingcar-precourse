# 자동차 경주 게임
## 진행 방법
* 숫자 야구 게임 요구사항을 파악한다.
* 요구사항에 대한 구현을 완료한 후 자신의 github 아이디에 해당하는 브랜치에 Pull Request(이하 PR)를 통해 과제를 제출한다.

## 과제 제출 과정
* [과제 제출 방법](https://github.com/next-step/nextstep-docs/tree/master/precourse)
# 기능 명세
## commit 메시지에는 (n-m)의 형식으로 기능 번호를 표시하였습니다.
## 예를들어, 1번 기능의 첫 번째 사항을 구현할 경우 (1-1)으로 표시됩니다.
1.	이동하는 자동차를 구현한다.
    1) 0~9 사이의 랜덤 값을 생성한다.
    2) random값이 4 이상일 경우 전진하고, 3 이하의 값이면 전진하지 않는 자동차를 구현한다.
    3) 자동차의 움직임을 저장하는 일급 콜렉션을 구현한다.
    4) (3)에서 구현한 일급 콜렉션을 자동차 클래스에 적용한다.
2.	사용자로부터 자동차 이름을 입력 받는다.
    1) 자동차 이름 클래스를 구현한다.
    2) 자동차 이름을 5자 이하로 제한한다.
    3) 사용자 입력에서 자동차 이름을 판별할 때에는 쉼표(,)를 기준으로 구분한다.
    4) 사용자에게 자동차 이름을 입력받는 모듈을 구현한다.
3.	사용자로부터 몇 번의 이동을 할 것인지 입력 받는다.
4.	턴을 진행할 때마다 주행 결과를 화면에 출력한다.
5.	게임이 종료되면 우승자를 판정한다.
6.	우승자 목록을 화면에 출력한다.
